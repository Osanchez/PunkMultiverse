# Interactables — stations, scanners, instruments

The three progression interactables share a shape: a `SavableComponent<Data>` whose Data
mutation is the durable, party-wide fact. The mod replicates each mutation as a reliable
event with a catch-up ledger (all in `ProgressionSync`).

## Station (the big one)

```
Station : SavableComponent<Station.Data>
  Data: allUpgrades (List<StationUpgrade> — copied from prefab at gen),
        installedUpgrades (IsUnlocked == Count > 0),
        connectedStations, lightPolygon, isFastTravelDestination, skipOpenAnim,
        emitDistressWhenLocked
        event UpgradeInstalled(Data, StationUpgrade)
  scene: platform (StationPlatform — the physical hatch: Open/Close/OpenImmediate),
         shop (Shop), unlockPrompt/shopPrompt, interactionCollider,
         enemyTrackingSystem (can't interact while enemies visible), enemyCollider

StationUpgrade : plain [Serializable] class — NOT a UnityEngine.Object!
  id (string — THE identity), cost, resourceUsed, activatedObject, animTriggerName,
  mapIconSprite, priceIncreaseMode/Amount
```

Interaction flow: `OnUseActivated` → locked? `TryInstallUpgrade(upgrades[0], unit)` (pay)
→ `Data.Install` → `UpgradeInstalled` subscribers: Station (open platform), ShipManager
(**respawn one dead ship at the station** — vanilla co-op revive), StationLightManager,
StationMapIcon, InGameMusicController. Unlocked? → `shop.StartShopping`.

| Aspect | Status | Mechanism |
|---|---|---|
| Unlock / upgrades | **EVENT** | postfix on `Data.Install(StationUpgrade)`, keyed by FNV-1a of `upgrade.id` (0.1.84 — the old `.name` capture never fired: plain class has no name). Applied on entity DATA so spectators/far players don't wait for stream-in. |
| Dead-player respawn | **EVENT**-derived | `RouteStationRespawn` prefix on `ShipManager.OnUpgradeInstalled`: net games revive the LOCAL ship only (vanilla's "first dead ship" could pick a gated puppet); resurrect broadcasts via ship life events |
| Rejoin checkpoint | **EVENT** | `LatestStationNetId`; disconnected players get `NeedsStationRespawn` → spawn at newest station |
| Shop purchases (player buys module) | per-player | purchaser pays own economy; module changes ride ModuleGridSync |
| Upgrade PRICES | **GAP** (known) | `RegisterStationUpgradePurchase` runs only on the purchaser → `GetTimesStationUpgradePurchased` diverges → other machines display/charge stale prices. Needs a purchase-count event if it matters. |
| Platform open animation | **EVENT**-derived | `Station.Bind` restores open state on stream-in (`skipOpenAnim`/`OpenImmediate`) |

## Scanner

```
Scanner : SavableComponent<Scanner.Data>   Data: areaId (byte), isUsed
```
Use → reveals a map area (`MapDrawer` zoom sequence). Generated by `ScannerGenerator` /
`ScannerAreaGenerator`.

| Aspect | Status | Mechanism |
|---|---|---|
| Use/reveal | **EVENT** | `ScannerUsedMsg` on `OnUseActivated` postfix; remote apply re-invokes `OnUseActivated(null)`, fallback marks `isUsed` |
| Catch-up | ledger | `UsedScanners` snapshot for late join/rejoin |

## Instrument

```
Instrument : SavableComponent<Instrument.Data> — discoverables (InstrumentDiscoverable assets)
```

| Aspect | Status | Mechanism |
|---|---|---|
| Discovery | **EVENT** | `InstrumentUsedMsg` (netId + discoverable name hash); replay invokes `Discover` with the local ship |
| Catch-up | ledger | `UsedInstruments` snapshot |

**Shared pattern for anything new here:** capture at the Data-mutation choke point,
identify by stable string id hash, guard with `_applyingRemote`, dedup via a ledger that
every machine accumulates (host-migration-proof), queue in `Pending*` if the entity isn't
resolvable yet, and apply on DATA rather than the streamed GameObject.
